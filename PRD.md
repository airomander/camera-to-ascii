# PRD: Webcam → ASCII Art Desktop Application

## 1. Резюме (Executive Summary)
Десктопное приложение, которое в реальном времени преобразует видеопоток с веб-камеры в ASCII-графику и отображает в отдельном окне. Проект спроектирован модульно для лёгкого расширения функционала.

## 2. Цели (Goals)
- Захват видео с веб-камеры (OpenCV)
- Преобразование каждого кадра в ASCII-символы по яркости пикселей
- Отображение результата в отдельном окне (pygame) с плавным FPS
- Производительность: минимум 15 FPS на среднем железе
- Модульная архитектура для лёгкого добавления фич

## 3. Не-цели (Non-Goals)
- Не веб-приложение (только десктоп)
- Не мобильное приложение
- Не video-to-file на первом этапе (сохранение — позже)

## 4. Архитектура

```
┌─────────────┐    ┌───────────────┐    ┌───────────────┐
│  webcam.py   │───▶│ascii_conv.py │───▶│pygame_display │
│  (захват)    │    │(конвертация)  │    │(рендер)       │
└─────────────┘    └───────────────┘    └───────────────┘
       │                   │                    │
       ▼                   ▼                    ▼
   OpenCV              numpy               pygame + SDL
```

### Схема данных
```
BGR frame (numpy) → grayscale → resize (120×n) → brightness map → char indices → render
```

### Компоненты
| Модуль | Ответственность | Технология |
|--------|----------------|------------|
| `camera/webcam.py` | Захват кадров, управление устройством | OpenCV |
| `converter/ascii_converter.py` | BGR→gray, resize, brightness→char | numpy + OpenCV |
| `renderer/pygame_display.py` | Окно, отрисовка, обработка ввода | pygame |
| `config.py` | Все настройки в одном месте | — |
| `main.py` | Оркестрация, главный цикл | — |

## 5. Технические решения (принятые)

| Решение | Выбор | Обоснование |
|---------|-------|-------------|
| Язык | Python 3.10+ | Быстрая разработка, богатый экосистема |
| Вывод | Отдельное окно (pygame) | Плавный рендер, контроль FPS |
| Ширина ASCII | 120 символов | Хороший баланс деталей и читаемости |
| Цвет | Ч/Б (v1), цвет (v2) | Упрощение первой версии |
| Шрифт | Courier New 12px | Моноширинный, доступен на всех ОС |
| Палитра | `@%#*+=-:. ` (9 градаций) | Классическая, проверенная временем |

## 6. Пользовательские сценарии

### Основной сценарий
1. Пользователь запускает приложение (python main.py)
2. Открывается окно с ASCII-представлением видео с камеры
3. Пользователь видит себя в реальном времени в виде ASCII-символов
4. Выход по ESC или закрытию окна

### Edge Cases
- Нет камеры → сообщение об ошибке и завершение
- Потеря кадра → пропуск, продолжение цикла
- Окно закрыто → корректное освобождение ресурсов

## 7. Ключевые метрики
- FPS: ≥15 target, ≥30 ideal
- CPU: ≤ 30% одного ядра
- Memory: ≤ 200 MB
